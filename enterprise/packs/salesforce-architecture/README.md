# Enterprise Salesforce Architecture Knowledge Pack

## Purpose

The Enterprise Salesforce Architecture Knowledge Pack defines enterprise-approved architecture rules for Salesforce product teams. It extends the Enterprise Spec Framework with architecture-specific governance for layered design, platform patterns, scalability, maintainability, observability, deployment, and documentation.

## Scope

This pack covers layered architecture, trigger handlers, service layer, domain layer, selector pattern, repository pattern, event-driven design, configuration, scalability, maintainability, observability, deployment, and documentation.

## Rule Location

Rules are stored in:

```text
enterprise/rules/salesforce-architecture/
```

Rule IDs use the `SFARCH-` prefix.

## Ownership

Primary owner: Enterprise Architecture

Contributors:

- Salesforce Center of Excellence
- Platform Engineering
- Product Architecture
- Integration Architecture
- Observability / SRE
- Release Management

## Lifecycle

Rules are advisory, warning, or blocking according to delivery risk. Product teams may request exceptions through the enterprise exception process, but rule IDs are stable and must not be reused after retirement.
