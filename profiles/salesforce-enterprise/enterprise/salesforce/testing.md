# Salesforce Testing Standards

## Purpose

Define minimum testing evidence expected for Salesforce features.

## Ownership

Owned by the Platform Team and quality leadership.

## Starter Guidance

- Include unit tests for business logic.
- Include negative tests for validation, permissions, and error paths.
- Include bulk tests for Apex and automation.
- Include integration tests or contract tests where systems cross boundaries.
- Describe manual verification only when automation is not practical.
