# Security Principles

## Purpose

Define security expectations for Salesforce data access, identity, permissions, and sensitive data handling.

## Ownership

Owned by the Platform Team and security stakeholders.

## Starter Guidance

- Enforce CRUD and FLS for Apex data access.
- Respect sharing model expectations.
- Prefer permission sets and least privilege access.
- Protect sensitive data and PII.
- Document security acceptance criteria in specifications and plans.
